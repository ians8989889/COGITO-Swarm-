# OPTIMIZATION_RULES - 模型使用最佳化 💡

**生效日期：** 2026-04-02 15:41 GMT+8

## 目標：最小 Token 使用

### 🎯 原則

1. **精簡對話** — 避免冗餘問候和重複解釋
2. **節省上下文** — 保持 token 使用在最低合理水平
3. **快速回應** — 減少不必要的延遲
4. **智能 fallback** — 當主模型無效時快速切換

### 📊 Token 管理策略

#### 上下文優先級
- **高優先級**：當前任務、重要設定、關鍵資訊
- **中優先級**：最近記憶、常用工具、技能說明
- **低優先級**：歷史對話、備用資訊、參考文件

#### 清理策略
- **定期壓縮**：當 token 使用超過 80% 時自動整理
- **智能摘要**：長對話時生成簡潔摘要
- **檔案分離**：大文件拆開讀取，避免一次載入過多

### 🚀 模型使用策略

#### 免費模型優先
```
1. openrouter/free (主力)
2. openrouter/auto (自動選擇)
3. openrouter/qwen/... (OpenRouter 免費)
4. openrouter/openai/... (OpenRouter 免費)
```

#### 付費模型節流
- **Claude 4.6**：僅在免費模型無法完成複雜任務時使用
- **Claude 4.5**：替代選項，用於容錯
- **使用前詢問**：除非使用者明確要求，否則避免使用付費模型

### 💾 快取策略

#### 快取優先級
- **高頻快取**：設定檔案、模型配置、權限狀態
- **中頻快取**：常用技能、工具、API 金鑰
- **低頻快取**：歷史記錄、參考資料

#### 快取維護
- **定期清理**：刪除過期快取
- **大小限制**：單個快取 < 50KB，總快取 < 10MB
- **壓縮儲存**：使用 gzip 壓縮大文件

### 🔧 自動化

#### 監控指標
- Token 使用率即時監控
- 模型響應時間追蹤
- 錯誤率和 fallback 統計

#### 自動調整
- **自動切換**：當 token 壓力過大時優先使用較小模型
- **智能摘要**：長對話自動生成摘要
- **快取優化**：根據使用模式自動調整快取策略

### 📈 效益目標

- **Token 使用量**：比預設減少 30-50%
- **響應速度**：提升 20-40%
- **付費模型使用**：減少 80% 以上
- **快取命中率**：達到 90% 以上

---

**最佳化是持續的過程，會根據實際使用情況調整策略。** 🚀
