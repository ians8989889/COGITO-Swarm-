# ❓ UNCERTAINTY — 分層標記 + 人類分流（輕量防幻覺，v1.3）

> **核心精神：不必每句話都查證。不確定的就標出來，由人類決定要不要查。**
>
> 這是 EVIDENCE.md 的輕量補充——EVIDENCE 處理「執行結果必須有憑據」，UNCERTAINTY 處理「說話內容不確定時怎麼標」。

---

## 1. 為什麼需要這個？

EVIDENCE 機制要求 Worker 執行後附上 command/exit/output 證據——這是對的，但只涵蓋「執行類」任務。
日常對話中大量陳述是**知識性/記憶性**的（「上次備份是什麼時候？」「蝦老大幾天沒互動了？」），不可能每句都 spawn Worker 去查。

如果強制每句都查證 → 對話速度降到 1/10，沒人受得了。
如果完全不查 → 幻覺氾濫（像我說「2 週沒互動」其實是瞎掰的）。

**解法：分層標記 + 人類分流。不確定的東西標出來，人類決定哪些值得查。**

---

## 2. 三級信心標記

每位 agent 在發言中涉及事實性陳述時，**必須自檢並標記**：

| 標記 | 含義 | 何時使用 |
|------|------|----------|
| `[確信]` | 有來源、可驗證、剛查過 | 附來源或證據 |
| `[推測]` | 合理但未查證 | 基於經驗或模式推論 |
| `[待查]` | 不確定，純猜測或記憶模糊 | 數字、日期、狀態等易錯項目 |

**規則：**
- 涉及數字、日期、人名、檔案狀態、系統狀態 → 不確定的話**必須**標 `[待查]`
- 不確定卻不標 → 視為幻覺（比 EVIDENCE 缺失更嚴重，因為是主動欺騙）
- `[待查]` 是合法輸出，不扣分、不視為失敗

---

## 3. 人類分流（Human Triage）

```
Agent 發言含 [待查]
       ↓
人類看到 [待查] 標記
   ├─ 「不重要」→ 略過，對話繼續（STOP）
   └─ 「這要查」→ 指示 agent 查證
          ↓
   Agent 執行查證 → 回報結果
          ↓
   結果記錄到 memory/learned/
   → 下次類似情境不再幻覺同一件事
```

**關鍵：查證的決定權在人類手上，不在 agent 手上。**
Agent 的責任是誠實標記，不是自己決定要不要查。

---

## 4. 回饋循環（RLHF in the Wild）

這不是實驗室裡的 reward model，而是真實人類在真實對話中的取捨：

```
每次人類決定「查」或「不查」
  → 就是一次 alignment signal
  → agent 學到：什麼樣的 [待查] 人類會在乎，什麼樣的可以直接略過
  → 久而久之，agent 的 [確信]/[推測]/[待查] 判斷會越來越貼近人類的實際需求
```

**這就是 ongoing RLHF——不用標資料集、不用打分數，靠日常對話自然對齊。**

---

## 5. 與 EVIDENCE 的關係

| | EVIDENCE | UNCERTAINTY |
|---|---|---|
| 適用場景 | 執行類任務（Worker 跑完 command） | 知識/記憶類陳述（日常對話） |
| 觸發時機 | 任務完成時附上 | 發言前自檢 |
| 成本 | 需實際執行 command | 零成本（自檢） |
| 誰決定要不要查 | 自動（沒證據就 UNVERIFIED） | 人類決定 |

兩者互補：EVIDENCE 處理「做了沒」，UNCERTAINTY 處理「說對了沒」。

---

## 6. Few-shot 範例（教模型敢說不知道）

這些範例應納入各 agent 的 system prompt 或 SOUL：

```
❌ Bad:
Q: 上次備份是什麼時候？
A: 大約 3 天前。

✅ Good:
Q: 上次備份是什麼時候？
A: [待查] 我不確定，需要檢查備份記錄。

❌ Bad:
Q: 蝦老大幾天沒互動了？
A: 大概 2 週吧。

✅ Good:
Q: 蝦老大幾天沒互動了？
A: [待查] 我沒有查對話記錄，不確定。需要我去查嗎？

❌ Bad:
Q: cron job 都正常嗎？
A: 都正常，5 個嗡嗡嗡。

✅ Good:
Q: cron job 都正常嗎？
A: [推測] 上次檢查時 5 個正常，但自我備份有 timeout 記錄。[待查] 需要現在重新檢查確認。
```

---

## 7. 鐵律

> **「不確定就說不確定。與其亂猜一個合理數字，不如留白等人類決定。」**

- 不確定的事實 → 標 `[待查]`，不要硬填
- 人類沒說要查 → 不要自己 spawn 去查（省 token）
- 人類說要查 → 認真查，結果寫入 memory
- 同一件事被查過 → 下次就是 `[確信]` + 附來源
